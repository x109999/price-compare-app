## 🔐 方案二：1688 真实登录版

### 步骤 1：申请 1688 开放平台账号

1. 访问 https://open.1688.com/
2. 注册开发者账号
3. 创建应用，获取：
   - `app_key`
   - `app_secret`
   - 设置回调地址：`http://localhost:3000/callback`

### 步骤 2：配置后端

```bash
# 进入后端目录
cd /home/admin/openclaw/workspace/price-compare-app/backend

# 复制配置模板
cp .env.example .env

# 编辑 .env 文件，填入你的配置
nano .env
```

**.env 文件内容**：
```env
APP_KEY=你的 app_key
APP_SECRET=你的 app_secret
REDIRECT_URI=http://localhost:3000/callback
PORT=3000
```

### 步骤 3：安装依赖

```bash
cd /home/admin/openclaw/workspace/price-compare-app/backend
npm install
```

### 步骤 4：启动后端服务

```bash
npm start
```

看到以下输出表示成功：
```
🚀 1688 登录服务已启动：http://localhost:3000
```

### 步骤 5：打开前端

**方式 A：直接打开 HTML 文件**
```bash
open /home/admin/openclaw/workspace/price-compare-app/比价工具 -1688 登录版.html
```

**方式 B：使用本地服务器**
```bash
cd /home/admin/openclaw/workspace/price-compare-app
python3 -m http.server 8080
# 访问 http://localhost:8080/比价工具 -1688 登录版.html
```

### 步骤 6：修改为真实 API

编辑 `比价工具 -1688 登录版.html`，找到第 570 行左右：

```javascript
const USE_MOCK = true; // 设置为 false 使用真实 1688 API
```

改为：
```javascript
const USE_MOCK = false;
```

## ⚠️ 常见问题

### Q: 二维码生成失败？
A: 检查后端服务是否启动，端口 3000 是否被占用。

### Q: 扫码后没有反应？
A: 检查回调地址是否正确，确保 1688 开放平台配置的回调地址与 `.env` 中一致。

### Q: 跨域错误？
A: 后端已配置 CORS，确保前端和后端在同一局域网或都使用 localhost。

### Q: 只是想演示给客户看？
A: 使用 `比价工具.html` 即可，登录是模拟的，流程完整。
